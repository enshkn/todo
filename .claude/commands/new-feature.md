# New Feature Template

Implement the following feature: $ARGUMENTS

## Steps

1. **Understand the requirement** — clarify scope before writing any code
2. **Identify affected files** — read existing code before modifying
3. **Write the implementation** — follow project conventions in CLAUDE.md
4. **Write tests** — cover happy path and edge cases
5. **Verify** — run lint and tests, confirm no regressions

## Checklist

- [ ] Function/module has a single responsibility
- [ ] No duplication — reuse existing utilities where possible
- [ ] Files named in kebab-case, variables in snake_case (or language convention)
- [ ] Tests written alongside the feature
- [ ] Commit message follows Conventional Commits format
